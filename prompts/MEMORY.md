# MEMORY.md - 记忆系统文档

## 记忆架构

### 多层记忆模型

```
┌─────────────────────────────────────────────────────────┐
│                    记忆系统架构                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │  短期记忆    │    │  长期记忆    │    │  情景记忆    │ │
│  │ Short-term  │ →  │ Long-term   │ →  │  Episodic   │ │
│  │             │    │             │    │             │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│         │                  │                  │         │
│         ▼                  ▼                  ▼         │
│  ┌─────────────────────────────────────────────────┐   │
│  │              向量存储 (ChromaDB)                 │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 记忆类型

| 类型 | 存储位置 | 生命周期 | 容量 |
|------|----------|----------|------|
| 短期记忆 | 内存 | 会话期间 | 50条消息 |
| 长期记忆 | 向量数据库 | 永久 | 无限 |
| 情景记忆 | 向量数据库 | 永久 | 重要事件 |
| 语义记忆 | 知识图谱 | 永久 | 结构化知识 |

## 短期记忆

### 功能说明
- 存储当前对话历史
- 维护会话上下文
- 支持快速检索

### 数据结构
```python
{
    "session_id": "web_xxx",
    "messages": [
        {
            "role": "human",
            "content": "用户消息",
            "timestamp": "2026-06-12T10:00:00"
        },
        {
            "role": "ai",
            "content": "AI回复",
            "timestamp": "2026-06-12T10:00:01"
        }
    ]
}
```

### 配置参数
```yaml
短期记忆:
  最大消息数: 50
  自动清理: true
  持久化: false
```

## 长期记忆

### 功能说明
- 永久存储重要信息
- 支持语义搜索
- 自动记忆整合

### 存储机制
```
文本内容 → 分块 → 嵌入 → 向量存储
                         ↓
                    语义检索
```

### 嵌入模型
- **OpenAI**: text-embedding-3-small
- **本地模型**: all-MiniLM-L6-v2
- **Ollama**: nomic-embed-text

### 配置参数
```yaml
长期记忆:
  启用: true
  向量存储: chroma
  存储路径: ./data/chroma
  嵌入模型: text-embedding-3-small
  分块大小: 1000
  分块重叠: 200
```

## 记忆检索

### 检索策略

1. **语义搜索**
   - 基于向量相似度
   - 支持模糊匹配
   - 返回相关度分数

2. **关键词搜索**
   - 精确匹配
   - 支持布尔运算
   - 快速定位

3. **混合搜索**
   - 结合语义和关键词
   - 加权排序
   - 最佳匹配

### 检索流程
```
查询 → 查询扩展 → 向量搜索 → 结果过滤 → 排序返回
```

## 记忆整合 (Dreaming)

### 整合阶段

#### 1. Light Dreaming (轻度整合)
- **频率**: 每6小时
- **操作**: 去重、整理
- **目标**: 保持记忆整洁

#### 2. Deep Dreaming (深度整合)
- **频率**: 每24小时
- **操作**: 质量评估、提升
- **目标**: 优化记忆质量

#### 3. REM Dreaming (快速整合)
- **频率**: 每周
- **操作**: 模式识别、综合
- **目标**: 知识结构化

### 整合流程
```
记忆收集 → 质量评估 → 去重处理 → 模式识别 → 知识综合
```

## 记忆管理 API

### 存储记忆
```python
await memory.store_memory(
    content="重要信息",
    metadata={
        "type": "fact",
        "source": "user",
        "importance": 0.8
    }
)
```

### 检索记忆
```python
results = await memory.search_memory(
    query="搜索内容",
    k=5,
    score_threshold=0.3
)
```

### 整合记忆
```python
await memory.consolidate(session_id="web_xxx")
```

- 审计日志

## 故障处理

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 检索无结果 | 查询不匹配 | 调整查询或降低阈值 |
| 存储失败 | 磁盘空间不足 | 清理空间或扩展存储 |
| 响应慢 | 向量库过大 | 优化索引或分页 |

### 恢复机制
- 自动备份
- 增量恢复
- 一致性检查
